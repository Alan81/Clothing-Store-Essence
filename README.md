CEIN — Интернет-магазин одежды

CEIN — это полноценный frontend-проект интернет-магазина одежды с личным кабинетом, корзиной, списком избранного, фильтрацией каталога и формой оформления заказа. Проект демонстрирует навыки создания сложных клиентских веб-приложений с управлением состоянием и адаптивным дизайном.
Проект создан для портфолио frontend-разработчика и показывает умение работать с React-хуками, Bootstrap-компонентами, кастомным CSS и пользовательскими сценариями (корзина, избранное, заказы).


Какие проблемы решает этот проект

1. Управление несколькими состояниями пользователя:
   Проект решает задачу синхронизации трех независимых списков — корзины, избранного (wishlist) и истории покупок — в рамках одного приложения.

3. **Удобный интерфейс для онлайн-покупок:
    Пользователь может:
   - Изучать каталог с фильтрацией.
   - Добавлять товары в корзину с мгновенным отображением счетчика.
   - Откладывать понравившиеся вещи в избранное.
   - Оформлять заказ через форму.
   - Видеть свои прошлые покупки и сохраненные товары в личном кабинете.
4. Отсутствие перезагрузки страницы: Все действия (добавление в корзину, переход между разделами, фильтрация) происходят без перезагрузки — приложение работает как SPA с классической навигацией.
5. Адаптивность: Сайт корректно отображается на мобильных устройствах и десктопах, решая проблему кросс-платформенного UX.
6. Прозрачность для разработчика: Код организован с использованием кастомного хука `useCart()`, что позволяет централизованно управлять корзиной и избранным из любого компонента.

Инструменты и технологии разработки

1. Библиотека: React (функциональные компоненты, хуки.
2. Стилизация: Bootstrap 5, кастомный CSS.
3. Управление состоянием: React `useState`, `useEffect`, кастомный хук `useCart`.
4. Работа с DOM: `useRef` (измерение высоты для фиксированной шапки)
5. Адаптивность: Bootstrap-сетка, медиазапросы, `d-lg-none/d-lg-flex`

Практическая польза проекта

Для пользователей:
- Мгновенная обратная связь: При добавлении товара в корзину счетчик обновляется в шторке (navbar) без перезагрузки.
- Два способа сохранить товар: Корзина для покупок, избранное (wishlist) — для отложенного просмотра.
- Личный кабинет: История заказов + сохраненные товары — пользователь не забывает, что ему понравилось.
- Форма заказа: Оформление покупки с указанием цвета, размера и других характеристик.
- Большой фильтр в каталоге: Ускоряет поиск нужной одежды.

Для моего портфолио:
- Демонстрация умения реализовывать три взаимосвязанные сущности (корзина, избранное, заказы).
- Опыт работы с Bootstrap в React-приложении (навигация, коллапс-меню, адаптивные утилиты).
- Понимание кастомных хуков (`useCart`) для разделения логики и UI.
- Навык измерения реальных DOM-элементов через `useRef` (динамический отступ под фиксированную шапку).
- Реализация мобильной версии с разной структурой навигации для десктопа и телефонов.


Как можно развить проект

Ближайшие улучшения:
1. Подключить бэкенд(Firebase / Node.js + MongoDB):
   - Регистрация и вход пользователей (JWT-аутентификация)
   - Сохранение корзины и избранного в базе данных (чтобы не терялись после перезагрузки)
   - Реальные заказы с отправкой на email
2. Пагинация или бесконечный скролл в каталоге для оптимизации производительности.

Среднесрочное развитие:
4. Интеграция с платежной системой (Stripe / PayPal) — реальная оплата.
5. Админ-панель для управления товарами (CRUD операции).
6. Отзывы и рейтинги товаров (с модерацией).
7. Рекомендательная система ("Вам также может понравиться" на основе истории покупок).

Долгосрочное / Продвинутое:
8. Миграция на TypeScript — повысит надежность кода для команды.
9. Unit-тесты (Jest + React Testing Library) для критических сценариев (добавление в корзину, фильтрация).
10. PWA — установка на телефон как нативное приложение.
11. Чат поддержки в реальном времени (WebSockets).

- Мобильные (<992px): гамбургер-меню, компактные иконки, фильтр в выезжающем окне
